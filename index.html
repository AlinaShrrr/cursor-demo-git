Шпаргалка за командами гіта

  • git config --global user.name blabla - задати або змінити ім’я користувача
  • git config --global user.email blabal@bla - задати або змінити e-mail який прив’язаний до Git
  • git init - ініціалізувати порожній локальний Git-репозиторій
  • git status – перевірити стан
  • git add . – додати всі файли
  • git commit - покласти зміни в локальний репозиторій
  • git commit -m "Comment text here..."- Покласти зміни в локальний репозиторій і додати одразу коментар
  • git log – дивитися зміни які внесені в репозиторій
  • git checkout – зміни гілок та вивантаження їх вмісту у робочий каталог
  • git checkout branch_name - змінити гілку на branch_name
  • git checkout -b branch_name - Створити гілку на своєму локальному комп'ютері та перейти в цю гілку
  • git branch - Показати лише локальні гілки
  • git branch branch_name - Створити локальну гілку з назвою branch_name
  • git push - Покласти зміни з локального у віддалений репозиторій
  • git pull - Оновити дані (забрати зміни) з віддаленого репозиторію по поточній гілці
  • git merge - використовується для злиття однієї або декількох гілок у поточну
  • git clone – клонувати репозиторій що існує

Advanced

  • git cherry-pick - бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній гілці. Ця можливість корисна в ситуації, коли потрібно забрати парочку коммітів з іншої гілки, а не зливати гілку цілком з усіма змінами, що внесені до неї.
  • git rebase - це "автоматизований" cherry-pick. Він виконує ту ж роботу, але для ланцюжка коммітів, тим самим переносячи гілку на нове місце.
  • git revert - повна протилежність git cherry-pick. Вона створює новий комміт, який вносить зміни, протилежні до зазначеного комміту, по суті скасовуючи його.
  • git reset – відміна змін
  • git stash - використовується для тимчасового збереження всіх незафіксованих змін для очищення робочого каталогу без необхідності фіксувати незавершену роботу в поточній гілці.

gitignore файл

    –необхідний для приховування файлів і папок від системи контролю версій Git. Зазвичай приховують файли конфігурації (особливо з паролями), тимчасові файли і папки.
